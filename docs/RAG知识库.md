## 一、项目概况

信用卡周报自动化项目的 RAG 问答子系统已构建完成。用户可以通过自然语言提问，系统从历史公众号文章知识库中检索相关内容，借助 LLM 生成回答。

## 二、RAG 实现方案

### 2.1 检索方式：混合检索（BM25 + 向量检索 + RRF 融合）

| 组件 | 方案 | 说明 |
|------|------|------|
| 关键词检索 | BM25 (k1=1.5, b=0.75) | 经典概率检索模型，零外部依赖 |
| 语义检索 | BAAI/bge-small-zh-v1.5 | 中文 embedding 模型，512 维向量 |
| 融合算法 | Reciprocal Rank Fusion (RRF) | 基于排名的融合，避免分数尺度不一致 |
| 向量存储 | numpy 数组 + pickle 序列化 | 轻量级，适合小规模知识库 |
| 缓存 | pickle 序列化 | BM25: `data/bm25_cache.pkl`，向量: `data/vector_cache.pkl` |
| Top-K | 5 条 | 每条截取前 3000 字注入 LLM |

**混合检索优势：**
- BM25 擅长精确关键词匹配（银行名、卡名、专业术语）
- 向量检索擅长语义理解（同义词、近义词、跨期对比）
- RRF 融合两者优势，提升整体检索质量

**Fallback 机制：**
- 如果 sentence-transformers 不可用或模型加载失败，自动降级到 BM25-only 模式
- 通过 `/mode` 命令可手动切换 hybrid/bm25_only 模式

### 2.2 LLM 模型选择

| 模型 | 供应商 | 接入方式 | 状态 |
|------|--------|----------|------|
| llama-3.3-70b-versatile | Groq | `GROQ_API_KEY` 环境变量 | ✅ 已就绪 |
| qwen/qwen3-32b | OpenRouter | `OPENAI_API_KEY` 环境变量 | ✅ 已就绪（备选） |

**切换方式**（在 `rag_query.py` 第 23 行附近）：
```python
LLM_PROVIDER = os.environ.get("LLM_PROVIDER", "groq")  # 切到 openai 用 OpenRouter
```

### 2.3 Prompt 设计

System prompt 约束：
- 主要基于参考资料回答
- 参考文献不足时明确告知"未覆盖"
- 回答具体到活动名称、银行、时间
- 金额用 ¥ 符号
- 单条回答控制在 200 字以内
- 末尾标注 📌 参考依据

### 3.3 内容分类

| 分类 | 说明 | 对业务的价值 |
|------|------|------------|
| **持卡评判** | 各银行信用卡评分、推荐、对比 | ✅ **核心价值**：持卡建议评分时可直接引用 |
| **公告点评** | 银行公告解读（温暖升级、权益变更、退市） | ✅ **核心价值**：公告点评输出时引用历史同类事件 |
| **亮点挖掘** | 新卡亮点、活动亮点、值得关注的产品 | ✅ **核心价值**：亮点挖掘输出时引用历史对比 |
| 周报资讯 | 每周信用卡行业动态汇总 | ✅ 辅助背景 |
| 一般资讯 | 通用信用卡知识 | ✅ 辅助背景 |
| 知识科普 | 信用卡基础知识 | ✅ 辅助背景 |

### 3.4 对三项核心业务的价值评估

#### 持卡评判 —— ⭐⭐⭐⭐⭐ 高价值
- KB 中有大量历史评分文章（"XX卡评分"、"值得办的卡"）
- 可以支撑：某卡历史上怎么评价的？类似卡片评分对比？某银行发卡趋势？
- **示例查询**：`广发银行什么卡值得办` → 命中 3 条相关文章

#### 公告点评 —— ⭐⭐⭐⭐ 较高价值
- KB 中有 "缩水"、"温暖升级"、"退市"等公告解读
- 可以支撑：历史同类公告的处置方式？这次公告和上次比是更严还是更松？
- **示例查询**：`缩水公告` → 命中 3 条相关
- **注意**：部分"公告点评"分类的文章实际是被误分类的非卡内容，BM25 关键词检索可绕过此问题

#### 亮点挖掘 —— ⭐⭐⭐ 中等价值
- KB 中 "亮点挖掘" 分类有直接文章
- 但亮点挖掘需要跨期对比（这周的新卡和半年前的新卡比），KB 的 chunk 切分可能割裂上下文
- 建议：亮点挖掘场景需要结合最新周报内容为主，KB 为辅查历史

### 3.5 已知问题（清洗后）

1. **分类准确度**：部分文章分类可能不准确（如非卡内容被标为"公告点评"），BM25 关键词检索可绕过此问题
2. **chunk 边界**：500 字/段可能截断关联上下文，跨 section 的连贯内容可能被拆分
3. **时效降权**：2020 年之前的旧文章仍存在 KB 中，可能与当前卡政策不一致
