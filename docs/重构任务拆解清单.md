# 重构任务拆解清单

## 说明

本清单基于 [下一阶段重构蓝图](./下一阶段重构蓝图.md) 拆解，目标是把“从 Word 暴露的问题反推上游重构”的思路，落实成可执行任务。

优先级定义：
- **P0**：必须先做，直接决定后续重构是否站得住；对减少人工审核最关键
- **P1**：高优先级，P0 完成后应尽快推进；直接提升数据质量和输出稳定性
- **P2**：重要但可后置；主要用于收尾、瘦身、提升可维护性

状态建议：
- `未开始`
- `进行中`
- `已完成`
- `暂缓`

## 五、最小可落地版本建议

如果只做一个“最小闭环版本”，建议先完成以下 5 项：

- **P0-1** 扩 Schema
- **P0-2** 建立 normalizer 入口
- **P0-3** 输出 review_queue
- **P1-5** 统一分类器
- **P1-6** 统一来源识别

这样即使 Word 层暂时还没完全瘦身，你也已经能获得两个核心收益：
1. 同一条内容不会再多处各判各的
2. 人工可以优先只看异常条目，而不是通读全部内容

## 七、建议你现在就开始的前三项

### 1. P0 - 扩 Schema
因为所有后续能力都要有落点。

### 2. P0 - 建统一 normalizer 入口
因为不先统一入口，后面仍会双轨打架。

### 3. P0 - 产出 review_queue
因为这一步最直接减少人工审核成本。
