# Mode D：Markdown 合并、点评与整合

## 定位

Mode B 面向 Word 交付，输入重点是结构化 Word 条目；Mode D 面向 Markdown
编辑稿，输入可以是公众号整理稿、人工笔记或上游 Markdown 产物。

Mode D 默认采用三层结果：

1. **证据层**：按二级标题条目解析，保留原文正文、图片引用、来源、类别、优先级和点评。
2. **合并层**：用标题规范化键去重，审计 JSON 记录每条资讯来自哪些文件。
3. **编辑层**：生成类别统计、主题聚合、交叉点评和行动建议；原文仍在“逐条保留”章节中。

## 使用

运行：

python -X utf8 md_merge.py --input data\公众号文章整理_20260708.md data\公众号文章整理_20260711.md --output data\mode_d_merged.md

默认同时生成同名 JSON 审计文件。需要模型增强跨条目点评时追加 --llm；
LLM 不可用时仍保留本地规则产物。

## Mode E 发布稿

Mode E 读取 Mode D 成稿后，会重新组织为适合公众号发布的精简文章：

- 自动生成标题，也支持 --title 手工覆盖
- 将总量、分类数量和重点观察压缩到“本期速览”
- 隐藏来源、方法、行动建议、来源清单和内部声明
- 末尾集中生成“原文链接”模块
- 保留逐条资讯、点评、表格、列表、引用和图片占位

## 第一性原理

合并文档不应直接把全文拼接后让模型重写。正确的最小闭环是：

原始事实可追溯 → 条目去重 → 主题整合 → 点评/行动建议 → 可审核输出

这样可以避免模型在去重时误删事实、把不同银行规则混在一起，或把编辑判断伪装成来源事实。
