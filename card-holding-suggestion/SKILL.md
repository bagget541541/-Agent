# 信用卡持卡建议 — 基于标准 JSON 的分析引擎

## 任务目标
- **核心价值**：基于上游标准 JSON 批次数据，深度分析信用卡资讯，给出可操作的持卡策略建议
- **能力来源**：
  - 消费上游标准 JSON（CreditCardBatch 格式），无需手动输入
  - 通过 `analyze_batch.py` 脚本预提取关键评估要点
  - AI 根据结构化分析上下文 + 联网搜索，生成最终建议
- **能力范围**：
  - 新卡评估（是否值得申请、年费回报率）
  - 权益分析（变更利好/利空、是否影响持卡策略）
  - 活动参与（性价比评估、参与策略）
  - 销卡建议（哪些卡值得保留/建议注销）
  - 组合策略（多家银行卡片搭配方案）

## 数据流

```
上游 skill（news-analyzer / wechat-article-extractor）
  │  输出标准 JSON
  ▼
card-holding-suggestion / scripts / analyze_batch.py
  │  读取标准 JSON → 按银行/分类组织 → 预评分
  ▼
分析上下文 JSON / Markdown
  │  AI agent 读取 + 联网搜索
  ▼
最终持卡建议（文本输出 / 独立文档）
```

## 前置准备
- `analyze_batch.py` 无需额外依赖（仅使用 Python 标准库）
- 联网搜索需要 `requests` 库（已列在 `requirements.txt`）
- 标准 JSON 文件来自上游 skill 或手动归档

## 操作步骤

### 1. 预处理：从标准 JSON 提取分析上下文

```bash
python card-holding-suggestion/scripts/analyze_batch.py \
  --input data/batch_标准格式.json \
  --output analysis_context.json
```

**脚本参数**：

| 参数 | 必填 | 说明 |
|------|------|------|
| `--input` | 是 | 上游输出的标准格式 JSON 文件路径 |
| `--output` | 否 | 输出文件路径（不指定则打印到 stdout） |
| `--format` | 否 | 输出格式：`json`（默认，结构化）或 `markdown`（人类可读） |
| `--focus` | 否 | 只分析指定分类：`新卡` / `权益变更` / `活动` / `公告`（默认全部） |

### 2. 分析脚本的输出结构

`--format json` 输出包含：
- **category_summary**：按分类组织，每条附有预评估（评分、影响方向、建议）
- **banks**：按银行组织，附各类型条数
- 每条资讯的 **evaluation** 字段：
  - 新卡 → 自动评分（1-10）、亮点、适用人群
  - 权益变更 → 影响方向判断（正面/负面/无影响）
  - 活动 → 价值评分、风险提示、初步建议

`--format markdown` 输出可直接用于：
- 贴给 AI 助手作为上下文
- 人工阅读概要
- 作为报告附件

### 3. AI 分析生成建议

AI agent 读取预处理数据后，按以下框架输出建议：

#### 3.1 新卡是否值得申请

评估维度：

| 维度 | 评估要点 |
|------|----------|
| 年费回报 | 年费金额 vs 刚性权益价值，是否有减免条件 |
| 独特权益 | 是否具有不可替代的亮点 |
| 日常实用性 | 是否匹配持卡人的消费场景 |
| 持有成本 | 账户管理复杂度、刷卡要求 |

**判断标准**：
- 刚性年费卡：权益价值 > 年费，且日常用得上 → 建议申请
- 免年费/刷免卡：权益覆盖日常需求 → 建议持有
- 权益鸡肋、年费刚性、无独特价值 → 建议观望或放弃

#### 3.2 权益变更应对策略

| 影响方向 | 建议 |
|----------|------|
| 负面影响（缩水） | 评估是否仍值得保留、寻找替代卡片 |
| 正面影响（升级） | 确认续持、评估是否需要调整用卡习惯 |
| 无实质影响 | 维持现状即可 |

#### 3.3 活动参与建议

- **高价值活动**（评分 3-5）：叠加多卡、确定性收益、无坑 → 建议参加
- **中等价值**（评分 1-2）：需要一定投入，视情况参与
- **套路活动**：名额少、规则复杂、收益低 → 建议放弃

#### 3.4 综合持卡策略

基于本期所有数据，给出组合策略建议：
- 本期各银行卡片的表现排序
- 推荐申请/保留/注销的卡片清单
- 最优活动参与路线图

### 4. 输出建议框架

按以下结构输出：

```
## 【持卡建议】{批次标签}

### 🆕 新卡申请建议
- 强烈推荐申请：卡片名称 + 理由
- 值得关注：卡片名称 + 理由
- 建议观望：卡片名称 + 理由

### 🔄 权益变更提醒
- 利好：卡片 + 变更内容 + 建议
- 利空：卡片 + 变更内容 + 应对

### 🏷️ 活动参与推荐
- 必参加：活动 + 参与方式 + 收益估算
- 选择性：活动 + 条件 + 注意事项

### 📋 综合持卡策略
- 本期最优搭档组合
- 建议调整的卡片
- 风险提示（征信、年费陷阱、活动真实性）
```

## 注意事项

- **年费陷阱**：特别关注刚性年费卡，避免"被年费"
- **活动真实性**：部分活动为噱头，需识别名额限制和规则坑
- **联动权益**：一张卡注销可能影响整套持卡策略的收益
- **征信影响**：频繁申销卡会影响征信，建议做好规划后再行动
- **时效性**：信用卡权益和活动经常调整，建议给出建议时注明信息时效
- **联网搜索**：对于关键决策（刚性年费卡申请、大宗消费活动），建议补充联网搜索验证
- **预处理脚本只做初步评分，最终判断仍由 AI 综合市场信息后给出**
